4. Ant Design

5. `DailyStatistics` 类型未导出但被引用

结论：

`admin-web` 当前**不能通过生产构建**。

执行开发模式后，Vite 成功启动：

`http://127.0.0.1:3000`

2. 路由自动进入 `/dashboard`

4. WebSocket 连接失败

根本原因：

后端 `localhost:8080` 未成功启动。

### 4.2 backend 后端服务

技术栈识别结果：

1. Spring Boot 2.7.18
2. MyBatis-Plus
3. MySQL
4. JWT
5. WebSocket

#### 源码重新编译结果

执行：

```bash
mvn -DskipTests package
```

结果：

编译失败。

主要错误特征：

1. 多个实体类和 DTO 的 getter/setter 找不到
2. 多个 `log` 变量找不到
3. 大量报错呈现出 Lombok 生成代码未生效的特征

可能原因：

1. 当前本机使用 `JDK 25`
2. 该项目更适合在 `JDK 17/21` 下构建
3. Lombok 与当前编译环境兼容性存在风险

结论：

`backend` 当前**无法在本机现有 JDK 25 环境下顺利重新打包**。

#### 现成 jar 运行结果

目录中存在现成包：

`backend\target\jingqu-backend-1.0.0.jar`

直接运行 jar 后，应用启动失败。

失败原因：

MySQL 认证失败：

`Access denied for user 'root'@'localhost' (using password: YES)`

说明：

1. jar 本身至少可以进入 Spring Boot 启动流程
2. 当前主要阻塞点是数据库账号密码与本机 MySQL 不一致

### 4.3 android 游客端

技术栈识别结果：

1. Kotlin
2. Jetpack Compose
3. Gradle 8.7

#### Gradle 检查结果

执行：

```bash
.\gradlew.bat tasks --all
```

结果：

Gradle 启动失败。

直接原因：

`org.gradle.java.home=D:\jdk21` 配置无效。

说明：

`android\gradle.properties` 中写死了：

```properties
org.gradle.java.home=D\:\\jdk21
```

但当前机器上该路径不存在，因此 Android 工程尚未进入真正构建阶段就先被 JDK 路径卡住。

结论：

`android` 当前**无法直接在本机命令行构建**，需要先修复 JDK 路径配置。

### P1：数据库配置与本机不匹配

`backend\src\main\resources\application.yml`

当前写死：

1. 用户名：`root`
2. 密码：`123456`

实际测试中，本机 MySQL 不接受该账号密码。

### P1：后端源码在当前环境下无法编译

表现：

大量 Lombok 相关方法缺失。

优先怀疑：

1. JDK 版本偏新
2. 队友开发环境与本机不一致

### P1：Android 工程写死本地 JDK 路径

文件：

`android\gradle.properties`

问题：

`org.gradle.java.home=D:\jdk21`

该配置不具备可移植性。

### P2：admin-web TypeScript 质量未收敛

表现：

1. 类型声明缺失
2. 严格模式报错较多
3. 构建不通过

这说明项目更像“开发中原型”，还不是可稳定交付状态。

### P2：存在嵌套 Git 目录

目录：

`A5景区导览ai服务数字人\`

内部只有 `.git`，疑似误解压或残留仓库元数据。
